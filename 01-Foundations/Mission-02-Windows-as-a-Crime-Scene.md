# Mission 02: Windows as a Crime Scene

## Objective
Learn how a security analyst investigates a Windows system after a suspected security incident.

## Scenario
A user reports unusual activity on their computer. Your job is to treat the machine like a crime scene and collect evidence before making assumptions.

## Analyst Questions
- What happened?
- When did it happen?
- Which user accounts were involved?
- What files, processes, or applications were affected?
- Is the activity normal or suspicious?

## Evidence Sources
- Windows Event Logs
- User login history
- Running processes
- File changes
- Installed applications
- Network connections

## Skills Practiced
- Digital forensics mindset
- Log analysis
- Identifying suspicious behavior
- Documenting findings

## Outcome
By the end of this mission, you will understand how security analysts investigate a Windows machine and preserve evidence during an incident.
